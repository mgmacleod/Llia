Synths

There are three types of client-side synths:
  1. Instruments
  2. Effects
  3. Controllers

Technically these are instances of the Python SynthProxy class. There are also server-side synths running within SuperCollider. We are rarely directly concerned with the server side synths and unless otherwise noted the term "synth" refers to a client object.

The order in which synths are created is significant. In general an effect must be created prior to anything which serves as it's inputs.

If you wish to process an organ through a flanger and then a reverb, the reverb must be created first, then the flanger and finally the organ.

    
       [organ]-->[flanger]-->[reverb]-->
Creation Order:
  1. Reverb
  2. Flanger
  3. Organ

As always there is an exception. Instruments which use the Poly1 keymode may be created at any time. This is because Poly1 creates server-side synths on the fly which are therefore always newer the any other synth. This trick only works while using Poly1.

Control synths either generate or process control rate signals, and never directly process audio. The instrument and effect synths generate and/or receive audio rate signals. They may also receive control rate signals and in some cases generate control signals. Effects such as flangers often have internal LFOs or other control generates. These are often provided as control outputs of the effect.

The creation order for controller synths is not important, though there may be subtle differences.

Synths are "patched" together using buses in the Graph window. There are two types of buses:

  1. Audio Buses
  2. Control Buses

A bus may be connected to any number of inputs and outputs. The same is not true for synths, each synth input/output may only connect to a single bus.
