Keymodes

Previous( Bank )     Next( Group )     Home     Contents     Synth Catalog     

Each active synth has an associated keymode which manages how the synth responds to MIDI key events. Not every synth type is compatible with every keymode.

Currently there are six keymodes, each with it's own set of pros and cons. Additional keymodes may be added later.

  1. PolyN
  2. PolyRotate
  3. Poly1
  4. Mono1
  5. MonoExclusive
  6. Efx

PolyN is the default keymode. It instantiates a fixed number of voices ahead of time and then allocates them to play notes as needed. If the number of keys down exceeds the voice count then some previous notes will be stolen. When a new key down is received, PolyN first tries to use an un-allocated voice. If all voices are currently in use it steals the voice which has been held down the longest.

Pros:

  • Fast, no need to create synths on the fly.
  • Fixed resource load.
  • Portamento supported (in a fashion).

Cons:

  • Limited polyphony, voices are stolen if notes played exceeds voice count.

PolyRotate is similar to PolyN except that it allocates voices in strict rotation. In most cases PolyN is the better choice. PolyRotate shares the same set of pros and cons as PolyN.

Poly1 instantiates synths on the fly as required for incoming key events. The synths are destroyed once their key has been lifted and all envelopes have terminated.

Pros:

  • Simple
  • Up to 128 note polyphony (if resources are available).
  • May be used with any effects chain without regard to synth creation order.

Cons:

  • Complex synths may take too long to create on the fly.
  • Resource heavy if too many notes held, particularly with complex synths or long decay tails.
  • Effects which require some history between notes, such as portamento, do not work.

Mono1 is a monophonic keymode with last note priority. A single persistent server side synth object is created.

Pros:

  • Fast
  • Fixed resource load
  • Portamento supported

Cons:

  • Monophonic

MonoExclusive is a highly specialized keymode where only the initial key down is articulated. Once a key is down, additional keys are ignored until all keys have been lifted.

The intended use is to layer MonoExclusive with another keymode. This allows the first note of a legato passage to have an alternate or accented tone. A real world example is on certain antique electric organs where a click is produced only on initial key presses.

Efx is a specialized keymode for effects and control synths. As with mono mode a single persistent server side synth object is created. Efx mode passes the last keynumber, note frequency, and velocity to the effect, should it be interested. As long as any key is down the gate signal is high, if all keys are up the gate signal goes low.
