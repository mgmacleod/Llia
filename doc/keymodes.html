Keymodes

Home Contents Synth Catalog Previous: Group Next: Config

Each synth has a keymode which determines how it responds to MIDI note messages. The keymode is selected when the synth is created and it is not possible to change it later. Each keymode has a set of advantages and disadvantages which makes one keymode more suitable then another depending on its use. The following is a list of the currently available keymodes.

The term "voice" is used interchangeably with an active server-side synth running in SuperCollider.

  • PolyN. The PolyN mode creates a finite number of server side synths (voices) when the instrument is created. When a new note is played, PolyN first checks to see if there are any available voices. If so, one of the available voices is tasked with creating the new note. If no voices are available, then the voice which has been down the longest is stolen for the new note.
    • Pros:
      • Fast - voices do not need to be created on the fly.
      • Fixed resource load.
      • Limited portamento is supported.
    • Cons:
      • Complex
      • Limited polyphony
  • PolyRotate. Like PolyN, PolyRotate creates a fixed number of voices when the synthesizer is created. These voices are used in strict rotation. Generally PolyN would be the better choice.
    • Pros:
      • Fast - voices do not need to be created on the fly.
      • Fixed resource load.
      • Limited portamento support.
      • Simpler then PolyN.
    • Cons:
      • Limited Polyphony.
      • Not possible to hold a note while playing notes above it.
  • Poly1. Poly1 does not create any voices ahead of time. Instead voices are created on the fly as needed for incoming notes.
    • Pros:
      • Simple
      • Unlimited polyphony, at least if system resource allow it.
      • Poly1 instruments may be used with any effect without regard to the order the effect and synths were created.
    • Cons:
      • Complex synths may take too long to instantiate.
      • Resources can be exhausted, particularly with complex synths, or with long release tails.
      • Can not use effects which require continuity between voices, such as portamento.
  • Mono1. Mono1 is a monophonic mode with last note priority.
    • Pros:
      • Simple and fast, a single voice is created ahead of time and reused for all notes.
      • Fixed resource load
      • Supports portamento.
    • Cons: ?
  • MonoExclusive. MonoExclusive is a monophonic mode where only the first note of a legato passage is articulated.
    • Pros:
      • Simple
      • Fixed resource load
    • Cons: Highly limited application.
  • EFX. EFX is a specialized mode used only with effects and controllers. It passes key gate, last key number and velocity to the effects synth should it be interested in such things. In some ways it is like MonoExclusive, a single synth instance is created when the effect is created.
