Llia Installation

Home Contents Synth Catalog Previous: Home Next: Quick Start 1
Llia consist of two major compoentns:
  1. Python client application. Most user interaction ocures with the client.
  2. Code executing within SuperCollider.

The term server is overloaded. For simplicity the term "server" in this documentation refers to code running in SuperCollider as a whole. Unless otherwise noted it does not specifically indicate the SuperCollider server proper.

Copy the top level Llia folder to a convenient location. Henceforth this location will be called the <install> directory.

Install SuperCollider code:
  1. Copy contents of <install>/sc/Extensions/Classes/ to the SuperCollider extensions folder.
    • Linux:
      • system-wide /usr/local/share/SuperCollider/Extensions/
      • per-user ~/.local/share/SuperCollider/Extensions/
    • OSX:
      • system-wide /Library/Application Support/SuperCollider/Extensions/
      • per-user ~/Library/Application Support/SuperCollider/Extensions/
    • Windows:
      • ?
  2. From a terminal navigate to the directory <install>/sc/SynthDefs/
  3. Start the SuperCollider ide with the command: scide

    At this point the new Llia classes will be compiled into the SuperCollider library. Great care has been taken to eliminate any errors, however if an error should occur SuperCollider will fail to start properly. The only remedy is to either remove or repair the offending file.

  4. From the SuperCollider ide load and execute the file <install>/sc/SynthDefs/install_synths.scd

    install_synths.scd should be run each time new Llia synths are available.

Resolve python dependencies:

Llia requires the following Python modules:
  1. numpy
  2. pillow
  3. python-image-tk
  4. mido

    The mido package requires either rtmidi (preferred) or portmidi as a MIDI backend.

  5. For OSX only, install ActiveTcl (see tkinter with Tcl/Tk on MacOS)
Install client:
  1. Copy configuration file <install>/resources/config to the appropriate location:
    • Linux ~/.config/Llia/
    • OSX ~/.config/Llia/
    • Windows ?
  2. Edit above configuration file for the installed MIDI backend.
    Locate the "backend" value in the [MIDI] section and change it to read either.
    • backend : mido.backends.rtmidi
    • backend : midi.backends.portmidi
    Whichever is appropriate for your system.
  3. While editing the config file, make sure the following two lines appear under the [GUI] section:
    • gui : tk
    • no-splash : False
